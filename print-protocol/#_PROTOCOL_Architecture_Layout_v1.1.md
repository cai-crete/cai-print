# **[CAI CANVAS 통합 건축/디자인 저작 시스템 전체 프로토콜 v2.0]**

CAI CANVAS는 건축, 인테리어 및 디자인 프로젝트의 시각 자원을 입력받아, AI 기반 텍스트 자동 생성, 지능형 이미지 레이아웃 할당, 그리고 고해상도 양식(PDF/ZIP/Video)으로 자동 출력해 주는 통합 웹 기반 저작 도구입니다. 이 문서에는 해당 프로그램 내 모든 기능의 작동 원리와 표준 프로토콜이 기술되어 있습니다.

---

## 1. 전역 시스템 레이어 및 데이터 관리 프로토콜 (Data Management)
**1.1. 소스 데이터 수집 (Library & Upload)**
*   기본 제공(Default) 이미지 풀과 더불어, 사용자가 로컬 기기에서 선택한 수많은 이미지 파일들을 브라우저 내장 `FileReader`를 통해 Base64 텍스트 기반 소스로 인코딩하여 메모리에 적재합니다.
*   각 이미지에는 고유한 태그와 이름이 부여되며, 사용자가 특정 이미지에 왕관(Crown) 마크를 토글할 경우 해당 이미지는 `heroImage` State로 격상되어 향후 모든 배치에서 1순위 초강제 할당 권한을 갖습니다.

**1.2. 문서 설정 인터페이스 (Document Configuration)**
*   **Purpose(목적)**: Panel(전시 도판), Report(보고서), Drawing(사양서), Video(영상), Image(이미지) 중 목적 지정. 이 값은 후속 AI 텍스트 프롬프트와 레이아웃 구조를 완전히 뒤바꾸는 핵심 변수입니다.
*   **Orientation(비율)** 및 **Number of Pages(페이지 수)**: 렌더링될 실제 물리 격자 DOM의 사이즈(예: A0, A4, 1920x1080)와 반복될 페이지 렌더링 횟수를 결정합니다.

---

## 2. AI 텍스트 생성 및 데이터 파이프라인 프로토콜 (Text Generation)
**2.1. 시각 분석 및 프롬프트 주입 (Vision Analysis)**
*   선택된 이미지 중 상위 3장을 선별하여 `google/genai` (Gemini-3-Flash) 모델에 전송합니다.
*   문서 목적(Purpose)에 따라 AI에게 각각 다른 '페르소나(Persona)'를 부여합니다. 예를 들어 Panel은 '감성적인 아키텍처적 공간 설명'을, Report는 '기술적이고 서술적인 논리 전개'를 요청합니다.

**2.2. 구조적 마크다운 생성 및 사용자 개입 (Interactive Review)**
*   AI 모델은 '대제목, 소제목, 프롤로그, 메인 본문(단락별)' 등의 구조를 강제 수용하여 1차 마크다운 초안 텍스트를 작성하여 우측 UI 박스에 띄웁니다.
*   이 시스템은 'Human-in-the-loop' 방식을 지향하여, 기계가 만든 텍스트 초안을 사용자가 그 자리에서 직접 추가, 삭제, 윤문할 수 있는 틈(Review & Edit)을 제공합니다.

**2.3. JSON 슬롯 정규화 (Structured Slot Mapping)**
*   사용자의 컨펌이 끝난 텍스트 컨텍스트를 해당 템플릿(예: Cover, TOC, Body, Panel)이 요구하는 물리적 문자열 칸 수(Slot)에 정확히 맞추기 위해 2차 AI 구조 분해를 실행합니다.
*   템플릿의 박스별 시각적 붕괴(Overflow)를 막기 위해, 대제목(최대 12자), 메인 지문(최대 280자) 등 **글자 수 임계치 제어(Limits Rule)**를 가장 강력하게 적용하여 `[{title, text[]}]` 기반의 JSON 객체로 가공합니다.

## 3. 지능형 이미지 위계 분석 및 자동 레이아웃 프로토콜 (Intelligent Layout)
**3.1. 패턴 매칭 및 휴리스틱 스코어링 (Heuristics Tagging)**
*   파일 이름 등의 메타데이터를 정규식 알고리즘으로 스캔하여 뷰 포인트(`[BEV]`조감도, `[FPV]`투시도, `[PLN]`평면도, `[DIA]`다이어그램)를 자동차별 태깅합니다.
*   프로젝트 성격이 상업/주거/공공이냐에 따라 특정 파일(`[FPV]`, `[PLN]`)에 우선점수(+100) 보너스 가중치를 스코어링합니다.

**3.2. 슬롯 분배 (Slot Allocation Engine)**
*   **1순위 슬롯 (Hero Area)**: 왕관으로 강제 지정된 `heroImage`를 무조건 1순위 대형 슬롯에 넣고, 미지정 시 가중치가 가장 높은 조감도/주투시도 한 장을 자동 렌더링합니다.
*   **2순위 슬롯 (Info Area)**: 도면(`[PLN]`, `[ELV]`) 관련 자료 중 파일명 논리(순서)에 의거하여 나열 배치합니다.
*   **3순위 슬롯 (Support Area)**: 다이어그램이나 1순위 탈락 시각물들을 서브 그리드 칸에 채웁니다.
*   사용된 이미지가 DOM 박스 수보다 모자라면(Underflow), 해당 슬롯 인덱스에 빈 문자열(`''`)을 대입하여 에러 없이 White Space 마진 효과가 나타나게 디자인적 방어 로직을 띄웁니다.

---

## 4. AI 비디오 트랜지션 생성 프로토콜 (Video Generation)
**4.1. 이미지 시퀀스 추출**
*   템플릿 추출과 달리, 비디오 모드(`purpose === 'video'`)가 활성화된 채 GENERATE를 누르면 시각적 연속성이 필요한 이미지 리스트들을 취합합니다.

**4.2. 보간법 렌더링 처리 (Frame Interpolation - Replicate API)**
*   `google-research/frame-interpolation` 오픈소스 비전 AI 모델로의 서버 API 요청 체계를 구축하여, 두 장의 건축 렌더링 사이(A → B)를 초당 프레임 연산으로 메꿔 3D 카메라 줌인/아웃 필름 효과를 만드는 mp4 전환 비디오(Transition)를 자동 구축 및 반환합니다.

---

## 5. 최종 통합 익스포트 및 렌더링 프로토콜 (Export Engine)
HTML/CSS 기반의 실시간 편집 DOM 요소들을 고화질 출력물로 융합하고 클라이언트 드라이브에 저장하는 종착 시스템입니다.

**5.1. 오프스크린 렌더링 및 해상도 동기화 (Off-screen Sizing)**
*   출력을 위한 페이지 배열(`pageElements`)을 화면 밖에서 렌더링하되, A0 판넬(841x1189mm) 혹은 A3 도서와 같이 용도별 물리 규격에 정확한 픽셀 단위(`offsetWidth`, `offsetHeight`)를 강제 마운트시켜 캔버스 사이즈가 `0x0`으로 깨지는 것을 원천 봉쇄합니다.

**5.2. 고해상도 PDF 병합 알고리즘**
*   설정상 다중 페이지를 생성(예: 7페이지 보고서)했을 경우, `html2canvas`로 단일 이미지 조각을 추출한 뒤, `jsPDF` 엔진 내부 루프를 통해 페이지를 생성 보충(`addPage`)해가며 무손실의 **단일 PDF 문서** 구조로 연결하여 다운로드합니다.

**5.3. 이미지 일괄 폴더 체계 적용 (ZIP Engine & Fallback)**
*   PNG 또는 JPEG 옵션 하에 일괄 다운로드 요청 시, 무수한 페이지가 브라우저에 연속 다운로드 스팸(Spam)을 만들지 않도록, `JSZip` 모듈에 가상의 폴더(`[title]_images`) 메모리를 띄운 후 이미지를 묶어 압축 처리합니다.
*   라이브러리 패키지 문제나 브라우저별 `saveAs` 인터페이스 거부 현상이 일어나는 최악의 상황에서도 끊김 없이 로컬 전송되도록, `URL.createObjectURL(blob)`로 우회하는 Native 브라우저 강제 다운로드(Fallback) 방어 코드가 적용되어 있습니다.
