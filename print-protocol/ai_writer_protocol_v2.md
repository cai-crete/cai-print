# AI 작가(Writer) 프로세스 및 레이아웃 배치 프로토콜 V2.0

본 문서는 'No11. print' 시스템의 핵심 지능인 **건축작가 AI(Writer)**가 프로젝트 데이터를 수집하여 최종 리포트 레이아웃에 텍스트를 안착시키기까지의 모든 논리적 행위와 기술적 단계를 정의합니다.

---

## 1. 전 과정 시퀀스 (The Narrative Sequence)

AI 작가는 아래의 6단계 수직적 공정을 거쳐 리포트를 완성합니다.

### Step 1: 데이터 엠베딩 및 온톨로지 추출 (Contextualizing)
- **이미지 분석**: 이미지 라이브러리의 시각적 특성(매스, 보이드, 그리드, 재료, 빛의 방향 등)을 추출하여 건축적 언어로 번역합니다.
- **프로젝트 규격 파악**: 사용자가 선택한 리포트의 목적(건축 리포트 등)을 확인하고 톤앤매너를 결정합니다.

### Step 2: 서사적 위계 수립 (Hierarchical Structuring)
- **4단계 텍스트 위계**: `Report_template_31`의 물리적 공간에 맞춰 텍스트를 계층화합니다.
  - **Level 1 (Directing)**: 전체 프로젝트의 정체성을 담은 메인 타이틀.
  - **Level 2 (Theming)**: 페이지별 디자인 테마(concept, keyword)를 서브 타이틀로 도출.
  - **Level 3 (Summarizing)**: 페이지 전반을 통찰하는 가로형 요약문.
  - **Level 4 (Storytelling)**: 개별 이미지의 미적/기술적 가치를 설명하는 상세 스토리.

### Step 3: 메타 데이터 지능형 생성 (Meta-Data Generation)
- **키워드(Keyword) 자동 생성**: 프로젝트 맥락을 분석하여 최적의 4개 전문 키워드(예: URBAN GRID / POROSITY / NATURE-LINKED)를 생성합니다.
- **날짜 및 기업명 동기화**: 리포트 생성 당일의 날짜를 자동 생성하며, 기본 기업명(`CRE-TE`) 또는 사용자 커스텀 기업명을 메타 데이터로 관리합니다.

### Step 4: 언어적 정제 및 오류 교정 (Refining & Editing)
- **고종석의 오류교정**: 비문, 일본어식 조사, 불필요한 수식어를 제거하여 명료한 건축 문장으로 정제합니다.
- **브랜딩 격리**: 생성되는 모든 텍스트에서 'No11. print'와 같은 시스템 명칭 노출을 원천 차단합니다.

### Step 5: 물리적 영역 최적화 및 축약 (Physical Fitting)
- **영역 제한(mm)**: 템플릿별 텍스트 박스의 물리적 크기를 계산하여 글자 수를 자동 제한합니다.
- **이미지 매칭 예외 처리**: 물리적으로 이미지가 존재하지 않는 슬롯(Empty Slot)에 대해서는 설명을 생성하지 않고 빈 문자열(`""`)로 반환하여 시각적 오류를 방지합니다.

### Step 6: 목차(TOC) 동적 구성 및 최종 배치 (Final Placement)
- **동적 인덱싱**: 실제 생성된 페이지 수에 맞춰 목차 섹션을 유동적으로 구성하며, 유휴 항목은 삭제합니다.
- **넘버링(`01.`)**: 서브 목차에 번호를 자동으로 부여하여 가독성을 극대화합니다.
- **DOM 주입**: 생성된 JSON 데이터를 React 컴포넌트의 각 `editable` 슬롯에 1:1로 매핑하여 최종 렌더링합니다.

---

## 2. AI 작가(Writer)의 행동 원칙 (Core Principles)

1.  **공백 없는 완성 (Full-Fill Policy)**: 이미지가 있는 모든 텍스트 영역은 반드시 전문적인 내용으로 풍성하게 채웁니다.
2.  **데이터 무결성 (Integrity)**: 이미지의 수와 텍스트의 수가 일치하도록 매핑 지능을 가동합니다.
3.  **전문성 유지 (Professionalism)**: 단순한 미사여구가 아닌, '건축적 개념'과 '공간적 가치'를 중심으로 서술합니다.
4.  **편집 성향 존중 (Editability)**: 모든 결과물은 사용자가 즉시 수정할 수 있는 `editable` 상태로 제공됩니다.
