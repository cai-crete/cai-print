# 프로젝트 개발 히스토리 (HISTORY.md)

이 문서는 **CAI Canvas** 프로젝트 개발 과정에서의 주요 기술적 의사결정, 치명적인 오류 해결 사례, 그리고 사용된 핵심 프롬프트 패턴을 기록합니다.

---

## 1. 주요 의사결정 과정 (Major Decisions)

### 1.1 Library A: 결정론적 이미지 시퀀싱 (Forced Sequencing)
- **배경**: 초기에는 AI가 이미지 태그를 보고 자유롭게 배치하도록 했으나, 건축 보고서의 전문성을 위해 특정 순서(조감도 -> 평면도 -> 단면도 등)를 지켜야 한다는 요구사항이 발생했습니다.
- **결정**: Library A에 한해서는 AI의 자율성보다 **인덱스 기반 강제 매핑** 로직을 우선 적용했습니다. 이를 통해 어떤 이미지를 넣더라도 표준화된 고품질 보고서 레이아웃이 보장되도록 설계했습니다.

### 1.2 백엔드 서비스 통합 (`server.ts`)
- **배경**: 초기에는 기능별로(이미지 생성, 영상 보간 등) 서로 다른 포트와 서버 파일을 사용했습니다.
- **결정**: 유지보수 편의성과 프론트엔드 API 호출의 단순화를 위해 **모든 AI 로직을 `server.ts` (Port 3001)로 통합**했습니다. Gemini와 Replicate API를 한 곳에서 관리하게 되었습니다.

### 1.3 텍스트 자동 피팅 시스템 (`useAutoFitText`)
- **배경**: AI가 생성하는 텍스트의 길이가 일정하지 않아 고정된 템플릿 상자를 벗어나는 문제가 빈번했습니다.
- **결정**: 글자 수를 제한하는 대신, **실시간 폰트 스케일링 훅**을 개발했습니다. 컨테이너의 크기를 감시하여 글자가 넘치면 폰트 크기를 자동으로 줄여 시각적 완성도를 높였습니다.

## 2. 주요 오류 및 해결 방법 (Critical Issues & Solutions)

### 2.1 보고서 페이지 밀림 현상 (Cover Shift Error)
- **문제**: 표지(Cover)와 Hero 페이지가 첫 번째 이미지를 소비하면서, 이후 본문 페이지의 이미지 순서가 하나씩 밀리는(Off-by-one) 현상이 발생했습니다.
- **해결**: `handleGenerate` 로직에서 표지에 사용되는 이미지는 **'소비(Consume)' 처리하지 않고 참조만 하도록** 수정하여 본문 시퀀스가 정확히 `low angle view -1`부터 시작되도록 교정했습니다.

### 2.2 비디오 에셋 인식 오류
- **문제**: `video-example-0.5x`와 `video-example-1` 파일명이 혼용되어 영상 생성 시 잘못된 경로를 참조하는 문제가 있었습니다.
- **해결**: 템플릿 로직 내부에서 안정성이 검증된 **`video-example-1` 경로를 하드코딩**하여 인식률을 100%로 끌어올렸습니다.

### 2.3 패널 방향성(Orientation) 상태 누수
- **문제**: Report 모드에서 Panel 모드로 전환할 때, 이전의 가로/세로 선택 상태가 남아 있어 UI가 비정상적으로 출력되었습니다.
- **해결**: Purpose 버튼 클릭 시 **`orientation` 상태를 즉시 `null`로 초기화**하는 로직을 추가하여 사용자가 반드시 방향을 새로 선택하도록 강제했습니다.

## 3. 핵심 프롬프트 패턴 (Prompt Patterns)

### 패턴 1: 건축 전문가 페르소나 주입
> "당신은 전문 건축 비평가입니다. 제공된 이미지를 materiality(재료성), spatiality(공간성), vernacular(지역성) 관점에서 분석하여 제목과 3줄의 설명을 작성하세요."
- **효과**: 일반적인 설명이 아닌, 실제 건축 도서에 쓰일 법한 전문적인 용어와 톤앤매너를 유도합니다.

### 패턴 2: 엄격한 JSON 구조 강제
> "출력은 반드시 다음 JSON 스키마를 준수해야 하며, 기술적인 설명이나 서론/결론은 포함하지 마세요: `{ pages: [{ title: string, content: string }] }`"
- **효과**: 프론트엔드 데이터 파싱 오류를 방지하고 시스템 간 데이터 전송의 안정성을 확보합니다.

### 패턴 3: 제약 조건 기반 재구성
> "이전 텍스트의 내용을 유지하되, 글자 수를 150자 내외로 맞추고 가장 중요한 키워드 2개를 제목에 포함시키세요."
- **효과**: 레이아웃 공간 제약이 엄격한 템플릿에 맞춤형 콘텐츠를 생성할 때 사용합니다.
